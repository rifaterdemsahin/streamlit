## 🐛 Purpose

Real projects encounter real problems. This folder documents:

1. **Error logs** — what went wrong and when
2. **Near-misses** — problems caught before they caused damage
3. **Workarounds** — solutions that aren't perfect but work
4. **Lessons learned** — insights for next time

---

## 📝 How to Use

When you encounter an error or unexpected behaviour:

1. Add a new `.md` file describing the issue
2. Include: error message, context, root cause, fix applied
3. Tag it with the relevant stage (1–7)

Obsolete errors/workarounds → move to `_obsolete/` sub-folder. 🚮
